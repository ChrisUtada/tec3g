# 卡牌属性配置手册

> 所有配置在 Godot 检视面板中直接编辑，无需写代码。
> 编辑路径：`resources/cards/` 下任意 `.tres` 文件，选中后在 Inspector 中修改。

---

## CardData — 卡牌基础数据

**来源：** `scripts/card_data.gd`  
**面板位置：** 选中 `.tres` 文件后，Inspector 顶部区域

| 英文属性 | 中文名 | 类型 | 默认值 | 说明 |
|---------|-------|------|--------|------|
| `card_id` | 卡牌ID | String | `""` | 全局唯一标识，推荐格式 `"TYPE_name"`，如 `"ITEM_sdt"` |
| `card_name` | 卡牌名 | String | `"未命名"` | 显示在卡牌标题栏 |
| `description` | 描述 | String (多行) | `""` | 显示在卡牌正文区域 |
| `card_type` | 类型 | 枚举 | `ITEM` | ITEM 物品 / CHAR 人物 / CLUE 线索 / LOGIC 指令 / SCENE 场景 / DEBUFF 状态 |
| `tags` | 标签 | Array[String] | `[]` | 行为分类标签，用于查询和事件分发，如 `["item","gift"]` |
| `consumable` | 可消耗 | bool | `false` | 合成/使用后是否从场上移除 |
| `allow_duplicate` | 允许多张 | bool | `false` | 玩家能否同时持有多个相同卡牌 |
| `drop_once` | 仅掉落一次 | bool | `false` | 仅首次触发条件时掉落，之后不再掉落 |
| `bg_color` | 背景色 | Color | `(0.15,0.35,0.7)` | 卡牌主背景颜色 |
| `border_color` | 边框色 | Color | `(0.3,0.55,0.9)` | 卡牌边框颜色，也用于类型标签文字色 |
| `text_color` | 文字色 | Color | `(1,1,1)` | 标题文字颜色 |
| `icon` | 图标 | String | `""` | 显示在类型标签前的图标（建议 emoji） |
| `sense_type` | 感官类型 | 枚举 | `NONE` | 线索专用：NONE 无 / TASTE 味觉 / TOUCH 触觉 / SMELL 嗅觉 / HEARING 听觉 / VISION 视觉 |
| `recycle_value` | 献祭价值 | int | `0` | 拖入献祭卡时获得的因果律数量 |
| `interact_text` | 交互文本 | String | `""` | 选中/悬停卡牌时显示的提示文字 |
| `favorability` | 好感度值 | int | `0` | 人物卡当前好感度数值 |
| `favorability_drops` | 好感度掉落 | Array[FavorabilityDrop] | `[]` | 各好感度档位对应的掉落配置 |
| `corruption_time` | 腐化时间 | int (ms) | `0` | 存在指定毫秒数后自动触发腐化事件（`0` 为不触发） |
| `fatigue_trigger` | 疲劳触发 | bool | `false` | 放置到场上时是否触发疲劳计数 |
| `is_gift` | 初始礼物 | bool | `false` | 是否在游戏开局时自动赠予玩家 |
| `art` | 卡面贴图 | Texture2D | `null` | 可选卡面图片，拖入纹理资源即可 |
| `recipes` | 合成配方 | Array[CombinationRecipe] | `[]` | 堆叠组合配方列表 |

## CombinationRecipe — 合成配方条目

**来源：** `scripts/combination_recipe.gd`  
**面板位置：** 在 CardData 的 `recipes` 数组中 ➕ 新增

| 英文属性 | 中文名 | 类型 | 范围 | 默认值 | 说明 |
|---------|-------|------|------|--------|------|
| `target_id` | 目标卡牌ID | String | — | `""` | 与当前卡堆叠时触发合成的另一张卡的 ID |
| `result_id` | 结果卡牌ID | String | — | `""` | 合成产出的卡牌 ID（对应 `resources/cards/` 下的 `.tres` 文件） |
| `min_count` | 最少数量 | int | 1~100 | `1` | 一次合成至少产出几张 |
| `max_count` | 最多数量 | int | 1~100 | `1` | 一次合成至多产出几张 |
| `weight` | 概率权重 | float | 0.0~10.0 | `1.0` | 多个配方在同一目标下时，按权重随机选择。`weight=0` 表示不会触发 |

**注意：** 堆叠触发是双向的，A 的 recipes 或 B 的 recipes 只要有一方匹配即可。

## FavorabilityDrop — 好感度掉落条目

**来源：** `scripts/favorability_drop.gd`  
**面板位置：** 在 CardData 的 `favorability_drops` 数组中 ➕ 新增

| 英文属性 | 中文名 | 类型 | 范围 | 默认值 | 说明 |
|---------|-------|------|------|--------|------|
| `threshold` | 好感度档位 | String | — | `"low"` | 触发档位：`"low"` / `"medium"` / `"high"` |
| `result_id` | 掉落卡牌ID | String | — | `""` | 达到该档位时掉落的卡牌 ID |
| `min_count` | 最少数量 | int | 1~100 | `1` | 至少掉落几张 |
| `max_count` | 最多数量 | int | 1~100 | `1` | 至多掉落几张 |
| `weight` | 概率权重 | float | 0.0~10.0 | `1.0` | 同档位多个掉落时按权重随机选择 |

## 配置示例

### 人物卡 — 朱穗的好感度掉落

```
card_id: "CHAR_zs"
card_name: "人物：朱穗"
card_type: CHAR
favorability_drops:
  ├─ threshold: "low"    → result_id: "ITEM_handwritten_note", weight: 1.0
  ├─ threshold: "medium" → result_id: "ITEM_truename_clue",   weight: 1.0
  └─ threshold: "high"   → result_id: "ITEM_true_name",       weight: 1.0
```

### 合成 — 手电筒 + 观测

```
card_id: "ITEM_sdt"
card_name: "手电筒"
recipes:
  ├─ target_id: "LOGIC_observe", result_id: "ITEM_peek_truth", min: 1, max: 2, weight: 1.0
  └─ target_id: "LOGIC_observe", result_id: "ITEM_coin",       min: 1, max: 3, weight: 0.4
```

## 快速添加新卡牌

1. 在 `resources/cards/` 复制任一 `.tres` 文件，重命名为 `你的卡牌ID.tres`
2. 在 Godot 中选中新文件，在 Inspector 中编辑所有字段
3. 在 `main.gd` 加一行 `preload` + `spawn_card`（或后续用批量加载自动注册）
4. 如需新配方/新掉落，在对应卡的 `recipes` / `favorability_drops` 数组中 ➕ 新增
